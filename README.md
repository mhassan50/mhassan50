## 🔧 What I'm Building

**FredGPT — Natural Language → SQL Reporting Engine**
> Three-phase RAG pipeline: local Ollama model for routing + entity extraction → C# context assembler with fuzzy/phonetic matching → cloud LLM for SQL generation. Converts plain English questions into live restaurant analytics. Stack: Semantic Kernel, Ollama, Qdrant, Groq, ASP.NET Core.

**FRED Restaurant POS — Cross-Platform Ordering App**
> Full .NET MAUI POS app with tablet and mobile view variants, multi-level menu categories, cart, bill types, cash/card payment with bank discounts, KOT/ESC-POS receipt printing, and QR-based customer lookup. Integrated buy-N-get-1-free loyalty redemption ported from a legacy VB.NET WebForms system. Stack: .NET MAUI, ASP.NET Core, SQL Server.

**Fred KDS — Kitchen Display System**
> Android tablet/TV kitchen display built with .NET MAUI — 5-second SOAP polling for live order data, diff/merge update logic, 8-band color-coded prep timers over a 20-minute window, and per-branch/per-printer settings persisted as JSON. Stack: .NET MAUI, SOAP/XML, CommunityToolkit.Mvvm.

**Fred Concierge — Hotel Management & Customer Service App**
> Cross-platform concierge app for hotel operations and guest-facing customer service. Stack: .NET MAUI, ASP.NET Core.

**Fred Teams — Internal Company Teams App**
> Internal communication and operations app for company staff across branches. Stack: .NET MAUI, ASP.NET Core.

**Kitchen Performance Scoreboard (KPS)**
> CEO-facing live dashboard for a multi-branch restaurant group. BCG matrix visualization, branch-wise best sellers, trajectory reports, Google ratings integration, and role-based access control. Stack: Blazor Server, Radzen, Dapper, SQL Server.

**FRED Reports & Operations Portal**
> Internal Blazor Server app for operational requests, approval workflows, loyalty analytics, customer notifications, and segment sales reporting. Full role/permission system driven by a database table with custom auth state provider. Stack: Blazor Server, Radzen, Dapper, SQL Server.
